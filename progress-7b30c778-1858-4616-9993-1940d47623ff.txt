## Codebase Patterns
- Entity: Lombok @Data, @TableName, @TableId(type=AUTO) auto-increment
- Controller: @RestController + @RequiredArgsConstructor, Result<T> wrapper
- Validation: inline checks in controller, return Result.error(code, msg)
- Pagination: MyBatis-Plus Page<T>
- Cascade: CascadeService handles cascading deletes via mapper queries
- JWT: userId/role set as request attributes by JwtFilter
- Tests: @SpringBootTest + @ActiveProfiles("test"), @BeforeEach cleanup, mapper-based
- Service layer: interface + impl pattern; batch ops use @Transactional; joined data built via Map<String,Object>
- Grade patterns: teacher-scoped queries filter by opening->teacher chain; student queries filter by selection->student
- Logging: AOP @Aspect with @Around advice on controller methods
- Frontend: Vue3 + Element Plus + Pinia + Vue Router; axios instance with JWT interceptor and 401 auto-logout; role-based route groups (/admin, /teacher, /student); localStorage for token persistence; Layout.vue with sidebar menu per role
- Frontend admin CRUD pattern: api/<resource>.js with CRUD functions; view uses el-card + el-table + el-pagination + el-dialog; search input + add button in card header; edit/add share dialog with isEdit flag; delete with ElMessageBox.confirm showing cascade warning; form validation via el-form rules
- Frontend teacher pattern: teacher-specific API endpoints (/openings/mine, /grades/teacher); Dashboard doubles as profile page with password change; password change triggers logout+redirect to /login; batch grade input via table selection + dialog with el-input-number per row
- Frontend student pattern: selection API (/selections/available, /selections/mine, /selections/credits); Dashboard doubles as profile page with password change (same as teacher); Selections page shows credit summary card + available courses table + enrolled courses table with drop confirmation; Grades page shows grade list with average score tag
- Frontend API pattern: request.js axios instance; each resource gets api/<name>.js; role-specific queries use dedicated endpoints that filter by logged-in user's userId from JWT
- Frontend logs pattern: readonly table with inline filter form; no pagination (backend returns full list); filter params passed as query params
