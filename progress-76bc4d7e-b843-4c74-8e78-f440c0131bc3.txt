## Codebase Patterns
- Spring Boot 3.2 + MyBatis-Plus (must use mybatis-plus-spring-boot3-starter, not mybatis-plus-boot-starter)
- Tests use H2 in-memory DB with MODE=MySQL for compatibility
- Package: com.student.system
- Entities in entity/, Mappers in mapper/, Config in config/
- build.sh / test.sh wrap Maven commands
- schema.sql is auto-initialized by Spring Boot sql.init
- Admin user inserted via DataInitializer (CommandLineRunner) using PasswordEncoder, NOT schema.sql
- H2 doesn't support INSERT IGNORE — use DataInitializer for seed data instead
- BCrypt hashes in schema.sql may not match across JVM versions — always use PasswordEncoder at runtime
- JWT: jjwt 0.12.5 with io.jsonwebtoken.security.Keys for HMAC signing
- Auth: interceptor-based (not Spring Security filters), WebMvcConfig registers interceptor excluding /api/auth/**
- RequestContextHolder: ThreadLocal-based, cleared in afterCompletion
- Validation: spring-boot-starter-validation with Jakarta @NotBlank/@Size/@Min/@Max/@NotNull
- Test pattern: @SpringBootTest + @AutoConfigureMockMvc, admin ID fetched dynamically via JdbcTemplate
- Controller pattern: checkAdmin() helper, ResponseEntity<?>, errorBody() helper, /api/admin/{resource}
- Service pattern: @Transactional on write ops, toResponse() mapper method, Page<> return for list/search
- DTO pattern: {Entity}Request with validation, {Entity}Response with selected fields (no passwords)
- Cascade delete pattern: query child records, iterate and delete grandchildren, then delete children, then parent
- Course entity has no createdAt field — use null in response
- Test cleanup: DELETE FROM teacher WHERE name != 'admin' to preserve admin user for login
- Teacher tests: use PasswordEncoder bean to create BCrypt hashes in @BeforeEach
